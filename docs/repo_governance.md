# Repository Governance

This file defines canonical sources, derived artifacts, and update rules for `nightsearch-SAST`.

## 1) Canonical vs derived locations

### Canonical (hand-edited, source of truth)

- **Core runtime code**: `src/nightsearch_sast/**`
- **Config schema**: `src/nightsearch_sast/config.py`
- **Runnable configs**: `configs/default.yaml`, `configs/smoke.yaml`, `configs/real_example.yaml`
- **Primary user docs**: `README.md`
- **Planning board**: `reports/backlog.md`
- **CI behavior**: `.github/workflows/validate.yml`, `.github/workflows/validate-full.yml`

### Derived / generated (do not hand-maintain as source of truth)

- Metrics/artifacts under `artifacts/` (generated by scripts)
- Temporary processed data under `data/processed/` unless explicitly committed as tiny fixtures
- Planner snapshots such as `planner-repo-summary.md` (when generated)

### Archive / historical

- `reports/archive/**` for historical notes and superseded reports

## 2) Files that should not be hand-edited as primary interfaces

- Generated JSON metrics from runner scripts
- Generated NPZ outputs from data creation/conversion scripts
- Any one-off generated planning summary artifacts

## 3) Required consistency updates by change type

When changing key subsystems, update all linked canonical files in the same PR:

### Model/training logic (`src/nightsearch_sast/models/**`, `src/nightsearch_sast/training/**`)

Must check/update:
- `README.md` (if behavior/metrics/CLI outcomes change)
- `configs/smoke.yaml` (if shape/runtime assumptions change)
- relevant tests in `tests/test_smoke.py`
- CI only if command/runtime contract changed

### Data pipeline or real bundle contract (`src/nightsearch_sast/data/**`, conversion scripts)

Must check/update:
- `README.md` normalized NPZ contract section
- `data/README.md`
- `configs/real_example.yaml`
- `tests/test_smoke.py` real-data path tests

### Config schema (`src/nightsearch_sast/config.py`)

Must check/update:
- all affected files in `configs/*.yaml`
- scripts that read configs (`scripts/run_*.py`)
- tests that instantiate configs
- README config examples

### CLI/entrypoint behavior (`src/nightsearch_sast/main.py`, `scripts/run_*.py`)

Must check/update:
- README command examples
- CI commands if defaults or flags change
- smoke/integration tests covering changed flags

### Tests (`tests/**`)

Must check/update:
- marker usage (`slow` vs fast)
- workflow expectations in `.github/workflows/validate*.yml`

### CI/workflows (`.github/workflows/**`)

Must check/update:
- any docs that prescribe validation commands (`README.md`, `AGENTS.md` if needed)
- helper scripts invoked by workflows

## 4) Reports policy

`reports/` is mixed-use today (planning + milestone notes + archive). Target policy:

- `reports/backlog.md`: live planning board (canonical, machine-parseable table)
- `reports/archive/`: archived/superseded notes (non-canonical)
- Future recommended folder: `reports/milestones/` for dated snapshot reports

Naming guidance for new non-backlog reports:
- Prefer `reports/milestones/YYYY-MM-DD_<topic>.md` once folder exists
- Keep snapshot reports factual and tied to reproducible commands/configs

Archive rule:
- Move reports to `reports/archive/` when superseded, contradictory, or no longer part of active decision-making.
- Do not delete historical notes unless clearly redundant and approved.

## 5) Binary and generated artifact hygiene

- Avoid committing generated PDFs and bulky binaries.
- If an external reference PDF must be tracked, store it in an explicit non-root location (for example `docs/external/`) and document why it is versioned.
- Keep repository root focused on canonical project interfaces.
